Por que stacking venceu

Os pesos aprendidos pelo Ridge.

Em vez de escolher o melhor modelo individual, o stacking combina os 3 melhores com pesos aprendidos por regressão Ridge. Os modelos se complementam - KNN é bom em casos típicos, HGBR é bom em casos atípicos.

A complementaridade dos modelos

O ponto chave: sem o stacking, você teria que escolher 1 modelo. Com ele, o algoritmo decide quanto peso dar a cada um - e os erros se compensam em direções diferentes, baixando o MAE absoluto.

XAI · Importância das features

O que importa pro HistGBR v2.

Permutation importance: embaralha uma feature por vez e mede quanto o erro do modelo aumenta. Quanto mais aumenta, mais importante a feature. Target encoding do bairro (laranja) domina - o "pano de fundo administrativo" é o sinal mais forte.

Quando confiar

Confiabilidade da predição por tipo de setor.

XAI honesto: o modelo prediz para qualquer setor que receber, mas a confiabilidade depende do contexto. Para setores especiais (parques, militar, indígena, favelas com sigilo), a predição é estatística mas pode ser semanticamente questionável.

💡 Como ler os percentis (P50 / P75 / P90)

Quando o stacking tem P90 = 49,70%, NÃO significa que o modelo erra ~50% das vezes. Os percentis se leem assim:

  • P50 = 17,88% → metade das predições erra menos de 18%
  • P75 = 30,98% → 75% erra menos de 31%
  • P90 = 49,70% → 90% erra menos de 50%

Renda tem distribuição com cauda longa: alguns setores são intrinsecamente imprevisíveis (heterogeneidade local - condomínios de luxo ao lado de vilas operárias). Esses casos atípicos inflam os percentis altos. Para a maioria dos casos, o erro é ~18% - um resultado competitivo com a literatura de predição de renda por setor censitário.
